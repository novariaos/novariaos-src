# PCI Subsystem

The PCI subsystem provides device discovery and configuration for PCI/PCIe devices. Access goes through the legacy port-mapped I/O configuration mechanism (PCI Configuration Space mechanism #1).

## Configuration space access

PCI configuration space is accessed by writing to the address port `0xCF8` and reading/writing data through port `0xCFC`:

```
Address format (32-bit, written to 0xCF8):
  bit 31:    Enable bit (must be 1)
  bits 23:16 Bus number (0–255)
  bits 15:11 Device number (0–31)
  bits 10:8  Function number (0–7)
  bits 7:2   Register offset (DWord aligned)
  bits 1:0   Must be 0
```

## API

| Function                  | Description                                         |
|---------------------------|-----------------------------------------------------|
| `pci_read8(bus, dev, func, offset)`  | Read 1 byte from config space           |
| `pci_read16(bus, dev, func, offset)` | Read 2 bytes from config space          |
| `pci_read32(bus, dev, func, offset)` | Read 4 bytes from config space          |
| `pci_write16(bus, dev, func, offset, val)` | Write 2 bytes to config space     |
| `pci_write32(bus, dev, func, offset, val)` | Write 4 bytes to config space     |
| `pci_find_device_by_class(class, subclass, progif, ...)` | Scan all buses for a device |
| `pci_find_capability(bus, dev, func, cap_id, offset)` | Walk capability list      |
| `pci_enable_msi(bus, dev, func, addr, data)` | Enable MSI interrupts            |

## Common configuration space offsets

| Offset | Size | Field           | Description                          |
|--------|------|-----------------|--------------------------------------|
| `0x00` | 2    | Vendor ID       | PCI vendor (0xFFFF = no device)      |
| `0x02` | 2    | Device ID       |                                      |
| `0x04` | 2    | Command         | Bus master, MMIO enable              |
| `0x06` | 2    | Status          | Capabilities list present (bit 4)    |
| `0x08` | 1    | Revision ID     |                                      |
| `0x09` | 1    | Prog IF         | Programming interface                |
| `0x0A` | 1    | Subclass        |                                      |
| `0x0B` | 1    | Class Code      |                                      |
| `0x0E` | 1    | Header Type     | Bit 7 = multifunction device         |
| `0x10` | 4    | BAR0            | Base Address Register 0 (low 32-bit) |
| `0x14` | 4    | BAR1            | Base Address Register 1 (high 32-bit for 64-bit BAR) |
| `0x34` | 1    | Cap Pointer     | Offset of first capability           |

## Command register bits

| Bit | Meaning                       |
|-----|-------------------------------|
| 1   | Memory space enable (MMIO)    |
| 2   | Bus master enable (DMA)       |

The NVMe driver sets both bits at init:
```c
pcicmd |= (1 << 1) | (1 << 2);
pci_write16(bus, dev, func, 0x04, pcicmd);
```

## Device discovery

`pci_find_device_by_class` performs a brute-force scan of all 256 buses × 32 devices × 8 functions. For each slot it reads the class register at offset `0x08` and compares the class, subclass, and programming interface.

Example: find an NVMe controller (class=`0x01`, subclass=`0x08`, progif=`0x02`):

```c
uint8_t bus, dev, func;
bool found = pci_find_device_by_class(0x01, 0x08, 0x02, &bus, &dev, &func);
```

## Reading a 64-bit BAR

MMIO BARs may be 64-bit, in which case BAR0 holds the low 32 bits and BAR1 the high 32 bits. The bottom 4 bits of BAR0 are flags and must be masked off:

```c
uint32_t bar0_lo = pci_read32(bus, dev, func, 0x10);
uint32_t bar0_hi = pci_read32(bus, dev, func, 0x14);
uint64_t bar0    = ((uint64_t)bar0_hi << 32) | (bar0_lo & ~0xFull);
```

## MSI (Message Signalled Interrupts)

MSI replaces legacy INTx wires with in-band PCI writes. The driver writes a target LAPIC address and data word into the MSI capability structure so the device can signal interrupts by writing to memory.

`pci_enable_msi` walks the capability list looking for capability ID `0x05`, then programs:

| Field       | Offset in cap | Value                        |
|-------------|---------------|------------------------------|
| Message Address (low) | `cap+4` | `0xFEE00000` (LAPIC)   |
| Message Address (high)| `cap+8` | `0` (for 64-bit capable)   |
| Message Data  | `cap+12` (64-bit) or `cap+8` (32-bit) | IRQ vector |
| Control       | `cap+2`  | Enable bit set, multi-msg cleared |

The NVMe driver uses vector `0x41`.

## Known class codes

| Class | Subclass | Prog IF | Device type      |
|-------|----------|---------|------------------|
| `0x01`| `0x01`   | —       | IDE controller   |
| `0x01`| `0x08`   | `0x02`  | NVMe controller  |
| `0x02`| `0x00`   | —       | Ethernet         |
| `0x03`| `0x00`   | —       | VGA              |
| `0x06`| `0x00`   | —       | PCI Host bridge  |
