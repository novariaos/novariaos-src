# FAT32

FAT32 is a read/write filesystem used for persistent storage on block devices (e.g. IDE/ATA disks). It is the primary writable filesystem in NovariaOS.

## Disk layout

A FAT32 volume is divided into three regions:

```
+-------------------------+
| Reserved region         |  sectors 0 .. reserved_sectors-1
|  Boot sector (BPB)      |  sector 0
+-------------------------+
| FAT region              |  num_fats copies of the FAT table
|  FAT #1                 |
|  FAT #2                 |  (mirror)
+-------------------------+
| Data region             |  clusters 2, 3, 4, ...
|  Root directory         |  starts at root_cluster
|  Files and dirs         |
+-------------------------+
```

Data starts at:
```
data_start_sector = reserved_sectors + (num_fats × fat_size)
```

Cluster number → sector:
```
sector = data_start_sector + (cluster - 2) × sectors_per_cluster
```

## BIOS Parameter Block (BPB)

Key fields from the boot sector:

| Field                 | Description                            |
|-----------------------|----------------------------------------|
| `bytes_per_sector`    | Usually 512                            |
| `sectors_per_cluster` | Power of 2, determines cluster size   |
| `reserved_sectors`    | Sectors before the first FAT           |
| `num_fats`            | Number of FAT copies (usually 2)       |
| `fat_size_32`         | Sectors per FAT                        |
| `root_cluster`        | First cluster of the root directory    |
| `volume_label`        | 11-character volume label              |
| `signature`           | Must be `0xAA55`                       |

Minimum cluster count for FAT32: **65525**. Fewer clusters means FAT12/FAT16.

## FAT table

Each FAT entry is a 32-bit value (only the lower 28 bits are used).

| Value range          | Meaning                      |
|----------------------|------------------------------|
| `0x00000000`         | Free cluster                 |
| `0x00000002`–`0x0FFFFFF6` | Next cluster in chain   |
| `0x0FFFFFF7`         | Bad cluster                  |
| `0x0FFFFFF8`–`0x0FFFFFFF` | End of chain (EOC)      |

Both FAT copies are updated on every write.

## Directory entries

Each directory entry is 32 bytes. FAT32 supports two formats:

### 8.3 short name entry

| Field          | Size | Description                          |
|----------------|------|--------------------------------------|
| `name`         | 8    | Filename, space-padded               |
| `ext`          | 3    | Extension, space-padded              |
| `attr`         | 1    | File attributes                      |
| `fst_clus_hi`  | 2    | High 16 bits of first cluster        |
| `fst_clus_lo`  | 2    | Low 16 bits of first cluster         |
| `file_size`    | 4    | File size in bytes (0 for dirs)      |

### Long File Name (LFN) entry

LFN entries appear before the 8.3 entry in reverse order. Each holds up to 13 UTF-16LE characters. The last entry in the sequence has bit `0x40` set in the `order` field. Integrity is verified via an 8.3 name checksum.

Max filename length: **255 characters**.

## File attributes

| Bit | Constant           | Meaning          |
|-----|--------------------|------------------|
| 0   | `FAT_ATTR_READ_ONLY` | Read-only      |
| 1   | `FAT_ATTR_HIDDEN`    | Hidden         |
| 2   | `FAT_ATTR_SYSTEM`    | System file    |
| 3   | `FAT_ATTR_VOLUME_ID` | Volume label   |
| 4   | `FAT_ATTR_DIRECTORY` | Directory      |
| 5   | `FAT_ATTR_ARCHIVE`   | Archive        |

Combination `0x0F` (`READ_ONLY | HIDDEN | SYSTEM | VOLUME_ID`) marks a LFN entry.

## Implemented operations

| Operation  | Status      | Notes                                    |
|------------|-------------|------------------------------------------|
| mount      | done        | Reads BPB, validates signature           |
| unmount    | done        |                                          |
| open       | done        | Supports 8.3 and LFN lookup              |
| read       | done        | Traverses cluster chain                  |
| write      | done        | Extends chain on EOF                     |
| seek       | done        | Supports `SEEK_SET`, `SEEK_CUR`, `SEEK_END` |
| readdir    | done        | Returns both 8.3 and LFN names          |
| stat       | done        |                                          |
| mkdir      | done        | Creates `.` and `..` entries             |
| rmdir      | done        | Frees cluster chain                      |
| unlink     | not yet     |                                          |

## Cluster allocation

New clusters are found by scanning the FAT from cluster 2 upward. On allocation, the cluster is immediately marked as EOC. On deallocation (`free_chain`), each cluster in the chain is marked free in sequence.
