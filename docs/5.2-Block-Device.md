# Block Device Abstraction

The block device layer provides a unified interface between storage drivers and filesystems. Drivers register themselves as block devices; filesystems mount them by name.

## Device registry

Up to `MAX_BLOCK_DEVICES` (16) devices can be registered simultaneously.

Each block device has:

| Field          | Description                              |
|----------------|------------------------------------------|
| `name`         | Device name, e.g. `hda`, `ram0`         |
| `block_size`   | Size of one block in bytes               |
| `total_blocks` | Total number of blocks on the device     |
| `ops`          | Read/write function pointers             |
| `private_data` | Driver-specific state                    |

## API

| Function                  | Description                                      |
|---------------------------|--------------------------------------------------|
| `block_init`              | Initialize the registry (zero all slots)         |
| `register_block_device`   | Register a new block device                      |
| `find_block_device`       | Look up a device by name                         |
| `get_block_devices`       | Return pointer to the full device array          |

## Operations

```c
typedef struct {
    int (*read_blocks) (block_device_t* dev, uint64_t lba, size_t count, void* buf);
    int (*write_blocks)(block_device_t* dev, uint64_t lba, size_t count, const void* buf);
} block_device_ops_t;
```

- `lba` — logical block address (zero-based)
- `count` — number of blocks to transfer
- Returns `0` on success, negative error code otherwise

## Registered devices

| Name   | Driver  | Writable |
|--------|---------|----------|
| `hda`–`hdd` | IDE | no (pending) |
| `ram0`, ... | Ramdisk | no |
