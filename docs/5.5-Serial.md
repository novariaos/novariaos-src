# Serial Driver

UART serial driver for COM1. Used for kernel debug output.

## Port

| Constant | Value   | Description |
|----------|---------|-------------|
| `PORT`   | `0x3F8` | COM1 base   |

## Configuration

Initialized with the following settings:

| Parameter   | Value               |
|-------------|---------------------|
| Baud rate   | 38400 (divisor = 3) |
| Data bits   | 8                   |
| Parity      | None                |
| Stop bits   | 1                   |
| FIFO        | Enabled, 14-byte threshold |

A loopback test is performed during init — if the chip does not echo `0xAE` back, `init_serial` returns `1` (faulty hardware).

## API

| Function         | Description                                  |
|------------------|----------------------------------------------|
| `init_serial`    | Initialize COM1, returns `0` on success      |
| `serial_received`| Returns non-zero if data is waiting to read  |
| `read_serial`    | Block until a byte arrives, return it        |
| `is_transmit_empty` | Returns non-zero if transmit buffer is empty |
| `write_serial`   | Block until ready, then send one byte        |
| `serial_print`   | Send a null-terminated string                |

## Register map (relative to `PORT`)

| Offset | R/W | Description                         |
|--------|-----|-------------------------------------|
| `+0`   | R/W | Data register / divisor low (DLAB)  |
| `+1`   | W   | Interrupt enable / divisor high     |
| `+2`   | W   | FIFO control                        |
| `+3`   | W   | Line control (data format / DLAB)   |
| `+4`   | W   | Modem control                       |
| `+5`   | R   | Line status (`bit 0` = data ready, `bit 5` = transmit empty) |
