# The NVM bytecode specification

## Signature
**Required 4-byte header** at the beginning of each `.nvm` file.  
Format: `NVMx` where `x` is the standard version (0-255).  
Example: `NVM0` = `0x4E 0x56 0x4D 0x30`

## Data Types and Stack
- Stack operates on 32-bit signed integers
- Stack size is 4KB (1024 cells)
- All values are stored as 32-bit signed integers
- **Big-endian** byte order for multi-byte values in instruction stream
- Stack pointer (`sp`) points to the next free slot
- Frame pointer (`fp`) enables function stack frames

## Process State
Each process maintains:
- Instruction pointer (`ip`)
- Stack pointer (`sp`)  
- Frame pointer (`fp`)
- 16 local variables (`locals[16]`)
- Capabilities array (`capabilities[]`)
- Active/blocked status

## Instructions

### Basic (6 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x00` | `HALT` | Halt the process |
| `0x01` | `NOP` | No operation |
| `0x02` | `PUSH imm32` | Push 4-byte immediate value (big-endian) |
| `0x04` | `POP` | Remove top value from stack |
| `0x05` | `DUP` | Duplicate top stack value |
| `0x06` | `SWAP` | Swap top 2 stack values |

### Arithmetic (5 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x10` | `ADD` | a + b (pops 2, pushes result) |
| `0x11` | `SUB` | a - b (pops 2, pushes result) |
| `0x12` | `MUL` | a * b (pops 2, pushes result) |
| `0x13` | `DIV` | a / b (pops 2, pushes result, checks division by zero) |
| `0x14` | `MOD` | a % b (pops 2, pushes remainder) |

### Comparisons (5 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x20` | `CMP` | Compare a-b: pushes -1/0/1 |
| `0x21` | `EQ` | Equality test: pushes 1 if equal, else 0 |
| `0x22` | `NEQ` | Inequality test: pushes 1 if not equal, else 0 |
| `0x23` | `GT` | Greater than: pushes 1 if a > b, else 0 |
| `0x24` | `LT` | Less than: pushes 1 if a < b, else 0 |

### Flow Control (7 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x30` | `JMP addr32` | Unconditional jump to 4-byte address |
| `0x31` | `JZ addr32` | Pop value, jump if zero |
| `0x32` | `JNZ addr32` | Pop value, jump if not zero |
| `0x33` | `CALL addr32` | Push return address, jump to subroutine |
| `0x34` | `RET` | Return from subroutine (pop return address) |
| `0x35` | `ENTER locals` | Create stack frame: push old fp, set new fp, allocate N locals |
| `0x36` | `LEAVE` | Destroy stack frame: restore fp and sp |

### Memory Access (8 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x40` | `LOAD idx` | Load local variable (0-15) to stack |
| `0x41` | `STORE idx` | Store stack top to local variable |
| `0x42` | `LOAD_REL off` | Load from stack[fp + 1 + offset] |
| `0x43` | `STORE_REL off` | Store to stack[fp + 1 + offset] |
| `0x44` | `LOAD_ABS` | Load from absolute address (address on stack, requires CAP_DRV_ACCESS) |
| `0x45` | `STORE_ABS` | Store to absolute address (address and value on stack, requires CAP_DRV_ACCESS) |
| `0x37` | `LOAD_ARG off` | Load argument (stack[fp - 2 - offset]) |
| `0x38` | `STORE_ARG off` | Store to argument (stack[fp - 2 - offset]) |

### System (2 instructions):
| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `0x50` | `SYSCALL id` | Invoke system call (arguments on stack, return value pushed) |
| `0x51` | `BREAK` | Debugging breakpoint (logs and continues) |

## Stack Frame Convention

When using `ENTER`/`LEAVE` for function calls, the stack is organized as follows:

```
HIGHER MEMORY ADDRESSES (bottom of stack)
+--------------------------------------------------+
| ... previous stack contents ...                  |
+--------------------------------------------------+
| ARGUMENT N          (N-th parameter)             |  <-- fp - 2 - N
| ...                                               |
| ARGUMENT 1          (second parameter)           |  <-- fp - 3
| ARGUMENT 0          (first parameter)            |  <-- fp - 2
+--------------------------------------------------+
| RETURN ADDRESS      (where to continue after)    |  <-- fp - 1
+--------------------------------------------------+
| SAVED FRAME POINTER (previous fp)                |  <-- fp (current frame pointer)
+--------------------------------------------------+
| LOCAL VARIABLE 0    (first local)                |  <-- fp + 1
| LOCAL VARIABLE 1    (second local)               |  <-- fp + 2
| ...                                               |
| LOCAL VARIABLE N-1  (N-th local)                 |  <-- fp + N
+--------------------------------------------------+
| ... temporary stack values ...                    |  <-- sp (stack pointer)
+--------------------------------------------------+
LOWER MEMORY ADDRESSES (top of stack)
```

### Access Formulas

| What | How to access | Formula |
|------|--------------|---------|
| **Arguments** | `LOAD_ARG off` | `stack[fp - 2 - off]` |
| **Return address** | (pushed by CALL) | `stack[fp - 1]` |
| **Saved fp** | (pushed by ENTER) | `stack[fp]` |
| **Locals** | `LOAD_REL off` | `stack[fp + 1 + off]` |

### Example: Function with 2 args and 3 locals

```
Stack after ENTER 3:
HIGHER ADDRESSES
+-------------------------+
| arg 1                   |  (fp - 3)
| arg 0                   |  (fp - 2)
+-------------------------+
| return addr             |  (fp - 1)
+-------------------------+
| saved fp                |  <-- fp
+-------------------------+
| local 0 (uninitialized) |  (fp + 1)
| local 1 (uninitialized) |  (fp + 2)
| local 2 (uninitialized) |  (fp + 3)
+-------------------------+
| (empty for future use)  |  <-- sp
LOWER ADDRESSES
```

### Instruction Sequence Example

```assembly
; Function prologue
ENTER 3           ; Create frame with 3 locals

; Access first argument (arg 0)
LOAD_ARG 0        ; Push arg0 onto stack

; Access first local
LOAD_REL 0        ; Push local0 onto stack

; Function epilogue
LEAVE             ; Destroy frame
RET               ; Return to caller
```

## Error Handling
Instructions that encounter errors:
- Terminate the process with exit code -1
- Log warning via `LOG_WARN`
- Set process state to inactive
- Stack underflow/overflow checks on all stack operations
- Division by zero protection on `DIV` and `MOD`

## Process Creation
Two methods:
1. `nvm_create_process` — basic process with empty stack
2. `nvm_create_process_with_stack` — process with pre-initialized stack (used for argv passing)

Processes are identified by PID (0 to MAX_PROCESSES-1) and registered in procfs.
