# devfs

The device filesystem (`devfs`) is a virtual filesystem mounted at `/dev`. It provides file-based access to hardware and pseudo-devices.

## Devices

| Path           | Read                          | Write                          |
|----------------|-------------------------------|--------------------------------|
| `/dev/null`    | returns EOF                   | discards all data              |
| `/dev/zero`    | returns zero bytes            | discards all data              |
| `/dev/full`    | returns zero bytes            | always returns `ENOSPC`        |
| `/dev/urandom` | returns random bytes (ChaCha20) | returns `EACCES`             |
| `/dev/tty`     | stub                          | prints to screen via `kprint`  |
| `/dev/stdin`   | linked to fd `0`              | -                              |
| `/dev/stdout`  | -                             | linked to fd `1`               |
| `/dev/stderr`  | -                             | linked to fd `2`               |

## Fixed file descriptors

Some devices have reserved file descriptor numbers:

| FD | Device         |
|----|----------------|
| 0  | `/dev/stdin`   |
| 1  | `/dev/stdout`  |
| 2  | `/dev/stderr`  |
| 3  | `/dev/null`    |
| 4  | `/dev/zero`    |
| 5  | `/dev/full`    |

## Implementation

- Registered as a virtual filesystem (`VFS_FS_NODEV | VFS_FS_VIRTUAL`)
- Mounted at `/dev` during kernel init via `devfs_init`
- Supports up to 32 registered devices
- Random number generation uses ChaCha20 seeded with hardware entropy
