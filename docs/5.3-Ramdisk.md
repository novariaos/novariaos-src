# Ramdisk Driver

The ramdisk driver exposes a region of memory as a read-only block device. It is used to back filesystems loaded entirely into RAM (e.g. an ISO image copied to memory at boot).

## Characteristics

| Property     | Value                  |
|--------------|------------------------|
| Block size   | 512 bytes              |
| Access       | Read-only (`-EROFS` on write) |
| Backing      | Any pointer + size     |

## API

```c
void ramdisk_register(const char* name, void* image, size_t size);
```

- `name` — block device name (e.g. `ram0`)
- `image` — pointer to the memory region
- `size` — size in bytes; must be a multiple of 512

Registers the ramdisk as a block device. Total block count = `size / 512`.

## Read

`ramdisk_read_blocks(dev, lba, count, buf)` — copies `count × 512` bytes from `image + lba × 512` into `buf`. Returns `-EINVAL` if the range exceeds the image size.
