# SPAWN syscall

Spawns a new process from a file descriptor, with inherited capabilities.

| Field    | Value         |
|----------|---------------|
| Number   | `0x01`        |
| Requires | `CAP_FS_READ` |

## Stack input

Arguments must be pushed onto the stack in the following order before the syscall:

1. Argument strings — each string pushed as individual bytes (lowest address first), terminated by `0`
2. `argc` — number of arguments
3. `fd` — file descriptor of the `.nvm` bytecode file to execute

```
HIGHER ADDRESSES
+-----------------+
| 0               |  null terminator of last string
| char N          |
| ...             |
| char 0          |  start of last string
| 0               |  null terminator of first string
| ...             |
| char 0          |  start of first string
+-----------------+
| argc            |
| fd              |  <-- sp - 1 (top)
LOWER ADDRESSES
```

## Return value

Pushes the PID of the spawned process onto the stack, or `-1` on error.

## Behavior

- Reads bytecode from `fd` into memory
- Constructs the initial stack with `argc` and `argv` for the child process
- Creates the new process with `nvm_create_process_with_stack`
- Copies the parent's capabilities to the child (`caps_copy`)

## Example

```assembly
; Push argument string "hello" (null-terminated)
PUSH 0        ; null terminator
PUSH 'o'
PUSH 'l'
PUSH 'l'
PUSH 'e'
PUSH 'h'
PUSH 1        ; argc = 1
PUSH 3        ; fd of the .nvm file
SYSCALL 0x01  ; SPAWN
; PID is now on top of stack
```
