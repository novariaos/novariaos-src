# IDE/ATA Disk Driver

The IDE driver detects and registers ATA hard disks as block devices. It uses PIO polling ‚Äî no DMA or IRQs required.

## Channels and drives

The PC/AT IDE interface provides two channels, each with two drives:

| Device | Channel   | Role   | Base port | Control port |
|--------|-----------|--------|-----------|--------------|
| `hda`  | Primary   | Master | `0x1F0`   | `0x3F6`      |
| `hdb`  | Primary   | Slave  | `0x1F0`   | `0x3F6`      |
| `hdc`  | Secondary | Master | `0x170`   | `0x376`      |
| `hdd`  | Secondary | Slave  | `0x170`   | `0x376`      |

All four drives are probed at boot via `ide_init`. Drives that respond to IDENTIFY are registered as block devices.

## Registers

All registers are accessed relative to the channel base port:

| Offset | Name         | R/W | Description                          |
|--------|--------------|-----|--------------------------------------|
| `+0`   | `DATA`       | R/W | 16-bit data transfer                 |
| `+1`   | `ERROR`      | R   | Error flags after a failed command   |
| `+1`   | `FEATURES`   | W   | Command parameters                   |
| `+2`   | `SECCOUNT`   | R/W | Number of sectors to transfer        |
| `+3`   | `LBA_LO`     | R/W | LBA bits 0‚Äì7                         |
| `+4`   | `LBA_MID`    | R/W | LBA bits 8‚Äì15                        |
| `+5`   | `LBA_HI`     | R/W | LBA bits 16‚Äì23                       |
| `+6`   | `DRIVE`      | R/W | Drive select + LBA bits 24‚Äì27        |
| `+7`   | `STATUS`     | R   | Drive status                         |
| `+7`   | `CMD`        | W   | Command register                     |

### Status bits

| Bit   | Constant    | Meaning             |
|-------|-------------|---------------------|
| `0x80` | `BSY`      | Drive is busy       |
| `0x08` | `DRQ`      | Data transfer ready |
| `0x01` | `ERR`      | Error occurred      |

## LBA28 addressing

LBA28 addresses 28-bit logical block addresses ‚Äî up to **128 GiB** (2¬≤‚Å∏ √ó 512 bytes).

The address is split across four registers:

```
DRIVE register:  1110 [slave] [LBA bits 27-24]
LBA_HI:          LBA bits 23-16
LBA_MID:         LBA bits 15-8
LBA_LO:          LBA bits 7-0
```

The `DRIVE` register top nibble is `0xE` for LBA mode:
- bit 7: always 1
- bit 6: LBA mode (`1`)
- bit 5: always 1
- bit 4: drive select (`0` = master, `1` = slave)

The total sector count is read from IDENTIFY words 60‚Äì61 (32-bit little-endian value).

## Drive detection (IDENTIFY)

At init, for each drive:

1. Select the drive via `DRIVE` register (`0xA0` master / `0xB0` slave)
2. Clear `SECCOUNT`, `LBA_LO`, `LBA_MID`, `LBA_HI`
3. Send command `0xEC` (`IDENTIFY`)
4. Read `STATUS` ‚Äî if `0x00`, no drive present
5. Wait for `BSY` to clear
6. Check `LBA_MID` and `LBA_HI` ‚Äî non-zero means ATAPI (skip)
7. Wait for `DRQ`, then read 256 √ó 16-bit words
8. Extract sector count from words 60‚Äì61

## Read (PIO)

One sector at a time (512 bytes = 256 √ó 16-bit reads):

1. Wait for `BSY` to clear
2. Write `DRIVE`, `SECCOUNT = 1`, `LBA_LO/MID/HI`
3. Send command `0x20` (`READ SECTORS`)
4. Wait for `DRQ`
5. Read 256 words from `DATA` port
